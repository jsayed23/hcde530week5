C5: Data Analysis with Pandas
I loaded ten seasons of data, converted it into a pandas DataFrame, and filtered to plays where Seattle was on offense. On that subset I used groupby with aggregation to compute DK Metcalf's targets, yards per target, and EPA per target for each of his six Seahawks seasons (2019–2024), which produced a six-row table I can use to pick his most efficient season. For Question 2, I needed a drive-level answer rather than a play-level one because the question asks about red zone trips, not red zone plays. I rolled the data up by grouping on the combination of season, game, and drive identifiers, flagged drives that reached the red zone, and then took the mean of a 0/1 touchdown indicator to get the per-season TD conversion rate. Choosing groupby plus mean over a flat aggregate was deliberate: a single overall rate would have hidden the year-over-year trend the question is asking about.

C2: Code Literacy and Documentation
Every pandas operation in my script has a comment that explains what I am asking and what the answer means, not just what the code does. The file opens with a docstring that names the data source, the scope (Seattle offensive plays, 2016–2025), and the three MP1a questions the script answers, so a reviewer can orient before reading any code. One specific documentation decision: when filtering for DK Metcalf I used a substring match instead of an exact match, and the inline comment explains why: the underlying data sometimes stores receiver names with the first initial only and sometimes with both initials, so a substring match is more robust than equality. Naming the trade-off in a comment is the kind of "why, not just what" documentation that makes the script readable for someone other than me.
